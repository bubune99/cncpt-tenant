# v0 Generated Code

## Overview
This directory contains all code generated by v0.dev organized by session and feature.

## Session Naming Convention
```
session-XXX-[feature-name]/
```
- XXX: Sequential session number (001, 002, etc.)
- feature-name: Descriptive name of what was generated

## Session Structure

Each session folder should contain:

### Required Files:
1. **v0-prompt.md** - The exact prompt used in v0
2. **generated-code.tsx** - The raw output from v0
3. **extraction-notes.md** - What parts to use/modify
4. **integration-status.md** - Track integration progress

### Optional Files:
- **generated-components/** - Extracted individual components
- **modified-code.tsx** - Code after adding meta-tags
- **test-results.md** - Testing notes

## Integration Workflow

1. **Generate**: Use v0.dev with specific prompt
2. **Document**: Save prompt and raw code
3. **Analyze**: Note what to extract/modify
4. **Add Meta-tags**: Enhance with builder annotations
5. **Test**: Verify in development branch
6. **Integrate**: Move to main platform
7. **Update Status**: Mark as integrated

## Meta-Tag Template

When preparing v0 code for integration, add meta-tags:

```tsx
{/* @builder:component id="v0-component" source="v0" editable="true" */}
  // v0 generated code here
{/* @builder:end */}
```

## Best Practices

1. Always save the original v0 prompt
2. Keep raw output unmodified
3. Document all modifications needed
4. Test thoroughly before integration
5. Archive old sessions regularly
